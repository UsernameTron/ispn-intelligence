2. Analyzes all major account reconciliations (AR, AP, cash, debt, accruals)

**Time Required:** 5-8 minutes for complete close cycle analysis (vs. 6-8 hours manual review)

### Reconciliation Deep-Dive Analysis

**Request:**
> "Analyze AR aging reconciliation for March 2024. Verify accuracy, identify aged items, flag unusual patterns, create exception tasks for team follow-up."

**What Claude Does:**
1. Loads AR aging reconciliation spreadsheet from Filesystem MCP

4. Identifies aged receivables (30/60/90+ day buckets)
5. Flags large-dollar items exceeding materiality threshold ($50K+)
6. Detects unusual patterns (unexpected credits, duplicate invoices)
7. Creates Asana task for each exception with priority and assignment
8. Generates reconciliation verification report for Controller approval

**Time Required:** 2-3 minutes per reconciliation (vs. 30-45 minutes manual review)

## Best Practices

1. **Pre-Close Preparation**: Load and verify trial balances on T-Day to identify anomalies before reconciliation submission.

2. **Daily Reconciliation Review**: Analyze reconciliations as submitted (T+1) rather than waiting for batch review, enabling same-day exception resolution.

3. **Consolidation Verification Early**: Verify consolidation workpapers on T+2 to allow time for rework if needed, avoiding last-minute delays.

4. **Exception Task Management**: Create Asana tasks immediately when exceptions identified, ensuring clear accountability and deadline tracking.

5. **Audit Documentation Continuous**: Generate audit documentation throughout close process rather than post-close compilation.

## Common Questions

**Q: How does Claude verify reconciliation accuracy without access to subledgers?**
A: Claude verifies reconciliation mathematical accuracy, identifies aged items and unusual patterns, and cross-references to GL. Subledger verification remains accounting team responsibility, but Claude flags items requiring investigation.

**Q: Can Claude detect fraud or intentional errors?**
A: Claude identifies unusual patterns and exceptions that may indicate errors or fraud (unusual debits/credits, duplicate entries, unexplained adjustments). Investigation and determination remain Controller responsibility.

**Q: What if reconciliation data exists in different formats or systems?**
A: Claude analyzes Excel-based reconciliations stored in Filesystem MCP. If reconciliations exist in other formats, export to Excel template for analysis. Standardized templates improve analysis consistency.

**Q: How does Claude maintain audit trail for analysis performed?**
A: All analysis results are stored in Filesystem MCP with timestamp and version control. Analysis methodology is documented in audit package. Controller approval is documented with sign-off notation.

**Q: Can Claude assist with quarterly and year-end close processes?**
A: Yes. The same analysis applies to quarter-end and year-end close with additional complexity (footnote analysis, tax provision review, audit coordination). Year-end processes may require 6-8 days vs. 4-day monthly close.
